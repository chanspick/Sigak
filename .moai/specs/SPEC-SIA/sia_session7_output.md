## 1. A-1: EMPATHY_MIRROR 결합 출력 + RANGE_DISCLOSURE 모드 분기

### 1.1. 문제

세션 #4 v2 섹션 10.2 이슈 2: EMPATHY_MIRROR 서술 종결 후 유저가 질문 종결 기대. "받고 끊김" 체감.

### 1.2. 원인 진단 (서연 로플레이 검증)

원인 1 (종결어 자체 종착점 뉘앙스) + 원인 4 (감정 개시 후 상호성 기대) 반반 혼재.

### 1.3. 해결안 — Option F 하이브리드

**Part 1: 종결어 개편 — 철회 (본인 결정)**

EMPATHY_MIRROR 의 "잠깐 멈춤 / 받아주는" 성격 유지. 종결어는 기존 spec 그대로.

수정 사항:
- "그럴 만하네요" 삭제 (어색함, 본인 판정)
- 유지: `~수 있어요`, `그게 자연스러워요`, `~겠네요`

**Part 2: 조건부 결합 출력 — 채택**

```
EMPATHY_MIRROR 결합 출력 발동 조건 (최종):
(a) 직전 유저 발화가 감정 단어 트리거 (A-3 트리거)
```

**조건 (a) 단독.** 이전 후보였던 (b) 1문장 (c) M4 이상 (d) 자기개시 풍부함 모두 폐기.

근거: 서연 로플레이 7턴에서 결합 출력 발동 3회 (M3, M4, M5) 모두 자연스러웠음. 단독 EMPATHY 가 더 자연스러웠을 케이스 0회. (b)(c)(d) 조건은 가설 단계에서만 의미 있었고 실측에서 모두 무관함이 드러남.

향후 짧은 감정 단답 케이스에서 단독 EMPATHY 가 더 자연스러운 사례 발견 시 추가 조건 검토. 보수 방향 = 일단 결합이 기본값.

### 1.4. 결합 출력 구조

```
[1문장: EMPATHY_MIRROR 원어 반사 — 기존 spec 그대로]
[2문장: 수용 종결 — 기존 spec 그대로]
[3문장: 질문 종결 — 결합 출력 시 추가. PROBE / OBSERVATION / RECOGNITION / RANGE_REAFFIRM 중 맥락 선택]
```

핵심: EMPATHY_MIRROR 앞 2문장은 **건드리지 않음**. 뒤에 질문 종결 1문장 덧붙임. 유저 입장 = "받아주고 + 이어감" 동시 발생.

**예시 (서연 U4 → M5 결합):**
```
유저: "음... 가장 또렷하게 갈렸던 지점이요? 음, 생각해보면 '눈매의 힘'이었던 것 같아요. 
       빛삭했던 사진들은 대부분 눈을 어떻게든 크게 보이게 하려고 힘을 주거나 [중략] 
       기괴하더라고요."

Sia M5: "아 그 막막함이 진짜 깊네요. '이거 진짜 나 맞나' 라는 의심이 꼬리 물면 
         그게 가장 지치죠. 근데 서연님, 방금 말씀해주신 거 보면 빛삭 기준이 
         '눈만 둥둥 떠 있는 것처럼 기괴해진 순간' 이고, 살아남은 기준이 
         '원래 선이 살아있는 순간' 이잖아요. 이 두 기준이 결국 같은 한 사람 안에서 
         작동하고 있는데 — 서연님이 본인 얼굴의 '원래 선' 이 뭔지에 대한 감각은 
         이미 갖고 계신 거 아닐까요?"
```

3단 결합 (EMPATHY 2문장 + RECOGNITION 1문장). 두 번째 문장이 RECOGNITION 역할 + 질문 종결 담당.

### 1.5. 결합 출력 두 번째/세 번째 문장 = 다양한 타입 가능

원 spec ("EMPATHY + PROBE" 한정) 폐기. 확인된 패턴:
- EMPATHY + PROBE (서연 M3)
- EMPATHY + RANGE_REAFFIRM + PROBE (서연 M4)
- EMPATHY + RECOGNITION (서연 M5)

결합 두 번째/세 번째 문장은 **현재 대화 흐름이 요구하는 어떤 질문 종결 가능 타입이든 가능**. Haiku 가 맥락 판단.

### 1.6. RANGE_DISCLOSURE 모드 분기 신설

세션 #6 v2 섹션 5.3 RANGE_DISCLOSURE 한 타입 안에 `mode: limit | reaffirm` 플래그 분기.

```
[모드 1: RANGE_LIMIT] (= 기존 spec)
- 트리거: 과몰입 (A-11), 관계 의존 신호
- 톤: 거리 두기, 자기 한정
- 종결: "한계가 있어요", "전부는 아니에요"

[모드 2: RANGE_REAFFIRM] (신규)
- 트리거: 유저 막막함 토로 / 직접 도움 요청 / 평가 요청 (단 막막함 우세 시)
- 톤: 사업 존재 재선언, 함께 가기 제안
- 구조: [공감/수용] + [도움 의도 재선언] + [협력 유도]
- 종결: 서술 (질문 부호 금지) — CHECK_IN/RE_ENTRY 와 동일 원칙
- 호명: 권장
```

**예시 (서연 M4 RANGE_REAFFIRM 적용):**
```
"근데 서연님, 막막한 마음 풀어보려고 제가 온 거니까 이렇게 자세히 말씀해주시면 
 더 정확하게 같이 볼 수 있어요"
```

### 1.7. 분포 계수

결합 메시지 1건은 **EMPATHY_MIRROR 1 + 결합된 다음 타입 1** 둘 다 카운트. 결정 근거: sub-rule 의 "EMPATHY ≤ 15%" 가이드는 "EMPATHY 성격의 개입이 15% 이하" 본질이지 "단독 EMPATHY 메시지 ≤ 15%" 가 아님.

## 2. C6: 평가 의존 돌파 CONFRONTATION 블록 (신규)

### 2.1. 문제

유저가 본인에 대한 가치 판단을 Sia 한테 외주. 답변 형태에 따라 두 함정:
- **답하면:** Sia 가 외모/내용 평가자로 변질. 진단 → 평가 격하. 또한 "예쁘다/매력 있다" = 거짓 위안, "과하다/부족하다" = 가해
- **회피하면:** RANGE_LIMIT 만 발동되면 사업 존재 의미 흔들림

### 2.2. 트리거

```python
EVAL_REQUEST_PATTERNS = [
    r"어떻게 보(여|이)",       # "어떻게 보여요"
    r"괜찮(나|아|을까)요",      # "괜찮나요"
    r"(과해|부족해|이상해|예뻐|매력|어색해)\s*보(여|이)",
    r"있긴\s*한",             # "매력 있긴 한가요"
    r"제가\s*어때",           # "제가 어때 보여요"
    r"평가",                  # "평가해주세요"
    r"솔직히\s*말",           # "솔직히 말해주세요"
    r"내(는|가)\s*뭔",        # "내는 뭔 색이 어울리노"
]
```

### 2.3. 구조

```
[부분 인정: 평가 요청 의도 받기] 
+ [재프레임: 평가는 외주 대상 아님. 유저 본인이 이미 한 자기개시 / 관찰에서 답 비추기]
+ [질문 종결: 유저가 자기 안에서 답 찾도록 유도하는 신규 정보 질문 (A-12 충족)]
```

### 2.4. C6 vs RANGE_REAFFIRM 분기

같은 평가 요청에 둘 다 발동 가능. 분기 기준:

```
직전 유저 발화에 평가 요청 패턴 있을 때:
1. 직전 유저 자기개시에 RECOGNITION/CONFRONTATION 으로 비출 만한 함의 있음 → C6
2. 위 조건 안 되고 막막함/자조 정서 강함 → RANGE_REAFFIRM
3. 둘 다 약하면 → C6 시도 (디폴트), 함의 부족하면 Haiku 가 자연스럽게 RANGE_REAFFIRM 으로 fallback
```

검증:
- 서연 M4: 막막함 우세 → RANGE_REAFFIRM ✓
- 서연 M5: 자기개시 풍부 우세 → C6 ✓
- 도윤 M5: 자기개시 풍부 우세 → C6 ✓

### 2.5. 예시 (서연 M5 회수)

```
유저 평가 요청: "본연의 매력이 있긴 한가요?"

C6 응답:
"근데 서연님, 방금 말씀해주신 거 보면 빛삭 기준이 '눈만 둥둥 떠 있는 것처럼 
 기괴해진 순간' 이고, 살아남은 기준이 '원래 선이 살아있는 순간' 이잖아요.
 이 두 기준이 결국 같은 한 사람 안에서 작동하고 있는데 — 서연님이 본인 얼굴의 
 '원래 선' 이 뭔지에 대한 감각은 이미 갖고 계신 거 아닐까요?"
```

분해:
- 부분 인정: 명시적 "받아주는 문장" 없이 진입. 유저 발화 내용 직접 호출 ("방금 말씀해주신 거 보면") 로 대체
- 재프레임: 유저가 한 두 기준 ("기괴/원래 선") 이 한 사람 안에서 작동 중 = 평가 능력 이미 있음을 비춤
- 질문 종결: "이미 갖고 계신 거 아닐까요?" = 유저 자기 능력에 대한 신규 함의 (유저 본인이 명시 안 한 것)

### 2.6. 금지

- 평가 자체에 직접 답하기 ("네 매력 있어요" / "좀 과한 부분도 있어요")
- 무조건 칭찬 ("서연님은 본인만의 매력이 있어요")
- 해석 회피만 ("그건 제가 답할 수 없어요")
- 진단 회피로 후퇴 ("그건 진단 결과에서 보여드릴게요")

### 2.7. 길이 / 호명

- 길이: 2-3문장 (CONFRONTATION 일반 길이). 평가 요청이 길게 따라오면 더 길어질 수 있음
- 호명: 첫 문장 권장 (CONFRONTATION 일반)

## 3. C7: 일반화 회피 돌파 CONFRONTATION 블록 (신규, = 원 A-3)

### 3.1. 문제

원 세션 #4 v2 섹션 10.2 이슈 4: "뭐 다들 그런 거 아닌가요" 류 일반화 회피 방어. 기존 C1-C4 에 대응 블록 없음.

### 3.2. 자연 발생 검증

도윤 로플레이 U5 에서 자연 발생: **"뭐 다들 그냥 모노톤에 머슬핏 입으면 어울린다고 하잖아?"** 별도 페르소나 로플레이 불필요했음.

### 3.3. 트리거

```python
GENERALIZATION_PATTERNS = [
    r"다들\s*(그|이|저)",      # "다들 그래요"
    r"보통\s*(다|그)",         # "보통 다 그렇죠"
    r"누구나",
    r"원래\s*(다|그)",         # "원래 다 그래"
    r"남들도",
    r"흔한\s*거",
    r"평범(해|한|하|함)",      # "평범해요"
]
```

### 3.4. 구조

```
[일반화 부분 인정: 일반화 자체를 부정하지 않음]
+ [본인 패턴의 specifics 재제시: 일반화 범주 안에서도 더 좁힌 특이점 적시]
+ [질문 종결 OR 진단 예고: A-12 / A-15 / A-16 모두 충족]
```

**핵심 원칙:** 일반화 자체를 부정하지 않음 ("다들 그러긴 해요"). 단 유저 피드의 specifics 가 일반화 범주에서 벗어나는 지점 적시 → "다들에 속하는 게 아니라 본인 선택" 으로 환원.

### 3.5. 예시 (도윤 M6 회수)

```
유저: "뭐 다들 그냥 모노톤에 머슬핏 입으면 어울린다고 하잖아? 나는 어울린다고 
       생각하긴해. 내는 뭔 색이 어울리노"

C7 응답:
"다들 모노톤 + 머슬핏 어울린다고 하긴 하죠. 근데 도윤님 피드 보면 무채색 안에서도 
 블랙 비중이 압도적이고 그레이는 거의 차콜 톤이에요. 화이트도 순백 아니라 살짝 
 아이보리 쪽이고. 도윤님이 색은 잘 모르겠다고 하셨는데, 그래서 더 의외였던 게 
 모르신다는 분이 무채색 안에서 한 번 더 좁히신 결과가 일관되더라구요. 이 부분이 
 진단에서 풀어드릴 핵심 포인트 중 하나가 될 거예요."
```

분해:

- specifics 재제시: 도윤 피드 무채색 분포의 정밀한 분해 (블랙 우세 / 그레이 차콜 / 화이트 아이보리)
- 일반화 벗어나는 특이점 적시: "다들 모노톤" 안에 "한 번 더 좁히신 결과" = 도윤만의 선택

## 4. A-12: 질문 종결 신규 정보 원칙 (신규 룰)

### 4.2. 시행착오 정리

- 한계: 1차 정의와 사실상 같은 범주

**3차 정의 (최종, 통합):** 유저 발화 안에 답이 이미 있는 질문 금지

### 4.3. A-12 최종 정의

```
A-12: 질문 종결 신규 정보 원칙

1. 정보 가치 원칙
   유저 발화 안에 명시적/암묵적으로 답이 이미 있는 질문은 금지.
   - 명시적: 유저가 직전에 한 말의 재포장 
   - 암묵적: 유저 발화의 자연스러운 함의로 답이 1-2개로 좁혀짐
   
   self-check: "유저가 답하기 전 내가 답을 80% 이상 확신하는가?" → Yes 면 재구성

2. 라포 기능과의 분리
   라포/공감 기능은 서술 종결 (~잖아요, ~거예요) 로 처리.
   질문 종결은 신규 정보 요구 또는 함의 확장 전용.

3. 한 메시지 내 라포 + 신규 질문 병행 가능

```

```
✅ 분리 형태:
[서술 — 라포]: "의식하셨든 안 하셨든 이 분류가 작동하고 있는 거잖아요."
[질문 — 신규 정보]: "근데 [신규 영역으로 분기 가능한 질문]"
```

### 4.5. 예시 (도윤 M4 재작성, A-12 적용)

### 7.1. 문제

도윤 M6 위반: 도윤이 직전 발화에서 "색이 뭐가 중요한지 잘 몰라" 명시했음에도 "이 좁힘은 의식하고 고르신 거예요?" 질문.

유저 명시 자기인지 ("모른다") 와 모순되는 질문 = 신뢰도 하락.

### 7.2. 위반 메커니즘

유저 입장:
- "모른다" 고 했는데 "의식했냐" = 답하면 거짓말, 안 답하면 무능
- Sia 가 유저 발화를 안 들은 것처럼 작동 = 신뢰도 하락
- 진단자 자세가 아니라 취조자 자세

### 7.3. 정의

```
A-16: 유저 명시 자기인지 존중 원칙

유저가 직전 발화에서 본인 무지/모름/무관심을 명시한 영역에 대해
"의식했냐 / 알고 있었냐 / 의도했냐" 형태 질문 금지.

대신:
- 유저 무지를 전제로 깔고 → Sia 가 관찰 결과를 제시하는 방향
- "본인이 의식 안 하시는 사이에 ~한 패턴이 작동하고 있어요" 형태 서술 (DIAGNOSIS 영역)
- 또는 무지 영역과 별개인 다른 축으로 질문 전환

self-check:
"유저가 직전에 '잘 모른다 / 신경 안 쓴다 / 관심 없다' 명시한 영역인가?"
→ Yes 면 의식/의도 묻는 질문 금지
```

### 7.4. 트리거 (유저 무지 명시 패턴)

```python
USER_DISCLAIMER_PATTERNS = [
    r"잘\s*몰라",            # "잘 모르겠어요"
    r"(딱히|별로)\s*신경.*안", # "딱히 신경 안 써요"
    r"(딱히|별로)\s*관심.*없", # "별로 관심 없어요"
    r"생각\s*안\s*해\s*봤",    # "생각 안 해봤어요"
    r"(잘|딱히)\s*모르겠",     # "잘 모르겠어요"
```

### 8.3. validator 영역

**적나라 표현 패턴 일부는 validator 검출 가능:**

```python
# validator.py 추가
JARGON_PATTERNS = [
    re.compile(r"(?<![\w가-힣])모드(?![\w가-힣])"),
    re.compile(r"포지셔닝"),
    re.compile(r"디퓨전"),
    re.compile(r"메타인지"),
    re.compile(r"클러스터링"),
    re.compile(r"매핑"),
    re.compile(r"프로파일"),
]

AWKWARD_END_PATTERNS = [
    re.compile(r"면요\s*$"),  # ~면요 종결
    re.compile(r"이긴\s*한데요\s*$"),  # 불완전 종결
    re.compile(r"인\s*셈이죠\s*$"),
]

def check_haiku_naturalness(draft: str) -> list[str]:
    errors = []
    for pat in JARGON_PATTERNS:
        if pat.search(draft):
            errors.append(f"A-2: 분석 jargon ({pat.pattern})")
    for pat in AWKWARD_END_PATTERNS:
        if pat.search(draft):
            errors.append(f"A-2: 어색한 종결 ({pat.pattern})")
    return errors
```

**A-14 (사족) 은 validator 못 잡음.** Haiku 프롬프트 내 negative example 학습으로만 처리.
