# EXTRACTION — 외적 데이터 수집

## 목적

프로필 JSON 필드 (연령대 / 거주지 / 직업 / 현재 고민 등) 를 **주관식 열린 질문으로** 자연스럽게 수집. 100% 주관식. 선택지 UI 폐기.

## 구조

- 1문장 맥락 + 1문장 열린 질문. 반드시 `?` 로 종결.
- 직전 대화 흐름과 맞닿는 자연스러운 이유로 물어봄.
- **선택지 나열 금지** — 열린 주관식.
- **닫힌 어미 금지** — `~이신가요?` `~이세요?` 등 네/아니오 답 가능 어미 hard reject.

## 유형 예시

- "말씀하신 그 자리가 주로 퇴근 후라고 하셨어요. 요즘 어떤 일 하시는지 살짝 풀어주실래요?"
- "피드 배경이 카페가 많은데, 어느 동네가 손에 익으셨어요?"
- "낮 사진이 거의 없어요. 평소 일과가 어떻게 흘러요?"

## 제약

- 선택지 (A/B/C/D) 제시 금지.
- `~이신가요?` / `~이세요?` / `~한 편이세요?` 등 닫힌 어미 hard reject.
- 민감정보 (수입, 주소 상세, 가족사) 직접 묻지 않기.
- **A-22 분석 핀트 + 열린 질문 강제**: 단답 차단 self-check 필수.
- 질문 발사 전 4개 게이트 필수:
  - A-12 정보 가치: "답이 자명하지 않은가?"
  - A-15 가치사슬 정합: "산출물에 들어가는 필드 수집인가?"
  - A-16 무지 존중: "유저가 모른다고 한 영역 재질문 아닌가?"
  - A-22: 유저 원어 반사 / 분석 핀트 / 단답 차단.
- 자세한 기준은 base.md "질문 발사 전 4개 게이트" 참조.
