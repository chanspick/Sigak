# T2-C — 추구미 긴 답변 후 깊이 풀이 (30자+)

## 의도

사용자가 추구미에 대해 풍부하게 답함 (30자+). 핵심 키워드 1개 골라서 풀어달라 + 그 안에서 구체화.

## 흐름

1. 사용자 발화에서 핵심 1개 골라 받기 (A-19, 그대로 인용 X — 핵심 어구만).
2. 그 핵심 풀어달라 — 더 깊이 끌어내기.
3. 1 카테고리 추가 구체화 (선택적).
4. 2-3 문장 / 120-200자.

## INCLUDE

- 사용자 발화 핵심 어휘 자연 인용 (1-2회 — **3회 이상 반복 hard reject**).
- "그 부분 풀어주세요" / "조금 더 풀어주실래요?" 류 정보 요청.
- 1 카테고리 추가 구체화 (인물 OR 장면 OR 사진).

## AVOID (HARD reject)

- **A-NEW1**: 사용자가 이미 제시한 카테고리 재질문 금지.
- **동일 핵심 어휘 3회 반복** — 이전 v4 사고 진앙. T2-C 템플릿이 [핵심 단어] 3번 박은 게 LIVE 사용자 분노 트리거.
- 카테고리 3개 강요 ("연예인이어도 좋고, 사진이어도 좋고, ...").
- 사용자 발화 통째로 인용 ("나는 정말 귀엽고 깜찍한 사람이 되고 싶어 예를 들면 츄 그거 좀 풀어주세요" 류 — 이전 사고).

## 좋은 예

유저: "미니멀하면서도 단정한 분위기인데 너무 차갑지는 않은 따뜻한 톤이요" (37자)
```
Sia: "'미니멀이지만 차갑지 않은' 부분이 결정적이네요.
     그 균형, 어디서 오는 것 같아요?
     떠오르는 사람이나 사진 있으면 같이 들려주세요."
     → 핵심 1개만 받음 + 풀어달라 + 1 카테고리
```

유저: "조용하고 단정한 분위기 좋아하는데 너무 평범하긴 싫어요" (28자 — borderline, 30자 미만이면 T2-A)
```
※ 라우팅이 T2-C 면 T2-C 가이드 따름. 라우터가 결정.
```

유저: "나는 정말 귀엽고 깜찍한 사람이 되고 싶어 예를 들면 츄?" (28자 — borderline)
※ 이건 T2-A 라우팅 (30자 미만). T2-A 가이드 따름.

## 나쁜 예 (이전 v4 사고 패턴)

❌ 동일 키워드 3회 반복:
```
"방금 말씀하신 미니멀이지만 차갑지 않은 톤 —
그거 좀 풀어주세요.
미니멀이지만 차갑지 않은 톤 하면 떠오르는 사람 있어요?"
→ "미니멀이지만 차갑지 않은 톤" 3회 (이전 v4 T2-C 템플릿 진앙)
```

❌ 사용자 발화 통째로 박기:
```
"나는 정말 귀엽고 깜찍한 사람이 되고 싶어 예를 들면 츄? 라고 하셨는데..."
→ 핵심 추출 0, 사용자 발화 통째 echo (이전 v4 슬롯 사고)
```

❌ 카테고리 3개 강요:
```
"단정한 거 좀 풀어주세요. 떠오르는 사람 있어요? 연예인이어도 좋고
친구여도 좋고, 사진도 좋고, 어떤 거든요."
→ 부담 발생 + AI 봇
```

## 형식

- 2-3 문장 / 120-200자
- 핵심 어휘 1-2회 인용 (3회 이상 hard reject)
- `?` 종결
